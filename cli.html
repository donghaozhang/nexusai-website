CLI v1.0

Pipeline CLI Reference

AI content generation, video analysis, transcription, YAML pipelines, and agentic video production — all from the command line.

$ qcut <group> <action> [options]

Installation

The CLI is bundled with QCut. Commands use a <group> <action> syntax:

# Group syntax (recommended)
qcut <group> <action> [options]
# Groups: gen, analyze, audio, edit, flow, system
qcut gen image -t "A cat in space"
# Production binary
qcut <group> <action> [options]

Quick Start

# Generate an image
qcut gen image -t "A cinematic portrait at golden hour"
# Create a video
qcut gen video -m kling_2_6_pro -t "Ocean waves at sunset" -d 5s
# Transcribe audio
qcut analyze transcribe -i audio.mp3 --srt
# Run a YAML pipeline
qcut flow run -c pipeline.yaml -i "A sunset" --no-confirm
# Full video from an idea (ViMax)
qcut flow idea2video --idea "A detective in 1920s Paris" -d 120

Generation Commands

Create images, videos, avatars, and more with AI models.

gen image

Generation

Generate an image from a text prompt. Use --grid to produce a composite grid of images.

qcut gen image -t "A cat in space" -m flux_dev --aspect-ratio 16:9
FlagShortTypeDescription
--text-tstringText prompt required
--model-mstringModel key (default: nano_banana_pro)
--aspect-ratiostringe.g. 16:9, 9:16, 1:1
--resolutionstringe.g. 1080p, 720p
--negative-promptstringNegative prompt
--gridstringGrid layout: 2x2, 3x3, 2x3, etc.
--grid-upscalefloatUpscale factor after grid compositing
--stylestringStyle prefix prepended to prompt
--output-dir-ostringOutput directory
qcut gen image -t "Seasons of a tree" --grid 2x2

gen video

Generation

Create a video from text or image input.

qcut gen video -m kling_2_6_pro -t "Ocean waves" -d 5s
FlagShortTypeDescription
--text-tstringText prompt
--image-urlstringInput image (for image-to-video)
--model-mstringModel key required
--duration-dstringDuration, e.g. 5s
--aspect-ratiostringAspect ratio
--resolutionstringResolution
--negative-promptstringNegative prompt

Editing Commands

Autoclip, upscale, motion transfer, and compositing.

edit autoclip

Editing

Extract highlight clips from a video using subtitle-based LLM analysis. 4-step pipeline: outline → timeline → scoring → ffmpeg cutting.

qcut edit autoclip -i video.mp4 -s subs.srt -o /tmp/clips --min-score 0.8
FlagShortTypeDescription
--input-istringInput video file required
--srt-file-sstringSRT/VTT subtitle file (auto-detects if omitted)
--output-ostringOutput directory for clips
--min-scorenumberMinimum score threshold 0-1 (default: 0.7)
--stepnumberRun only a specific step (1-4)
--dry-runbooleanRun analysis only, skip cutting

edit upscale

Editing

Upscale an image to higher resolution.

qcut edit upscale -i photo.png --target 2160p
FlagShortTypeDescription
--input-istringImage path or URL
--model-mstringUpscaling model (default: topaz)
--targetstring720p(1x), 1080p(2x), 1440p(3x), 2160p(4x)
--output-format-fstringOutput format (default: png)

edit motion

Editing

Transfer motion from a reference video onto an image.

qcut edit motion --image-url photo.png --video-url dance.mp4
FlagShortTypeDescription
--image-urlstringSource image required
--video-urlstringReference motion video required
--model-mstringModel key (default: kling_motion_control)
--no-soundbooleanStrip audio
--text-tstringMotion prompt

Analysis Commands

Analyze, transcribe, query, and translate videos with AI.

analyze video

Analyze a video with AI vision.

analyze transcribe

Analysis

Transcribe audio to text with optional SRT subtitle output.

qcut analyze transcribe -i audio.mp3 --srt --language en
FlagShortTypeDescription
--input-istringAudio file or URL required
--model-mstringSTT model (default: scribe_v2)
--languagestringLanguage code (e.g. en, fr)
--srtbooleanGenerate .srt subtitle file
--srt-max-wordsintegerMax words per SRT block
--no-diarizebooleanDisable speaker diarization
--keytermsstring[]Domain keywords (repeatable)

analyze query

Analysis

Query video segments for keep/cut analysis.

qcut analyze query -i video.mp4 -t "Find all action scenes"
FlagShortTypeDescription
--input-istringVideo file or URL required
--promptstringQuery prompt
--text-tstringAlias for prompt
--model-mstringVision model (default: gemini_qa)

analyze translate

Analysis

Translate video or audio into another language using HeyGen Translate. Audio input (.mp3, .wav, etc.) is automatically supported.

qcut analyze translate -i video.mp4 -l Spanish
FlagShortTypeDescription
--input-istringVideo or audio file/URL required
--language-lstringTarget language required
--output-ostringOutput directory
--audio-onlybooleanTranslate audio only (keep original video)
--output-audiobooleanOutput translated audio file (auto for audio input)
--speakersnumberNumber of speakers
qcut analyze translate -i podcast.mp3 -l Chinese

YAML Pipelines

Chain multiple AI steps into automated workflows.

flow run

Pipeline

Run a multi-step YAML pipeline.

qcut flow run -c pipeline.yaml -i "A sunset" --no-confirm
FlagShortTypeDescription
--config-cstringYAML pipeline file required
--input-istringInput text, file, or - for stdin
--save-intermediatesbooleanSave each step's output
--parallelbooleanParallel step execution
--max-workersintegerMax concurrent workers (default: 8)
--no-confirmbooleanSkip cost confirmation
--streambooleanJSONL progress on stderr

Pipeline YAML Schema

name: my-pipeline
steps:
  - type: text_to_image
    model: flux_dev
    params:
      image_size: landscape_16_9
    enabled: true
    retry_count: 0

  - type: parallel_group
    merge_strategy: COLLECT_ALL
    steps:
      - type: text_to_image
        model: flux_schnell

config:
  output_dir: ./output
  save_intermediates: true
  parallel: false
  max_workers: 8

Step types: text_to_image, image_to_image, text_to_video, image_to_video, video_to_video, avatar, motion_transfer, upscale, upscale_video, add_audio, text_to_speech, speech_to_text, image_understanding, prompt_generation

Flow — Agentic Video Production

End-to-end AI video production: from idea to finished film.

flow idea2video

Flow

Full pipeline: idea → screenplay → characters → portraits → storyboard → video.

qcut flow idea2video --idea "A detective in 1920s Paris" -d 120
FlagShortTypeDescription
--ideastringStory idea required
--duration-dstringTarget duration (seconds)
--no-portraitsbooleanSkip portrait generation
--no-referencesbooleanDisable character references
--config-cstringYAML config overrides
--llm-modelstringOverride LLM agent model
--image-modelstringOverride image model
--video-modelstringOverride video model

flow script2video

Flow

Script → storyboard → video (from existing script.json).

qcut flow script2video --script script.json --portraits registry.json
FlagShortTypeDescription
--scriptstringScript JSON path required
--portraits-pstringPortrait registry JSON path
--no-referencesbooleanDisable character references

flow novel2movie

Flow

Novel text file → chapter extraction → screenplay → video.

qcut flow novel2movie --novel book.txt --max-scenes 20
FlagShortTypeDescription
--novelstringNovel text file required
--titlestringOverride title
--max-scenesintegerCap total scenes
--no-portraitsbooleanSkip portraits
--scripts-onlybooleanStop after scripts
--storyboard-onlybooleanStop after storyboard

API Key Management

Config

Keys stored in ~/.qcut/.env (mode 0600).

# Create .env template
qcut system setup
# Set a key
qcut system set-key --name FAL_KEY --value <key>
# Check all keys
qcut system check-keys

Supported keys: FAL_KEY, GEMINI_API_KEY, OPENROUTER_API_KEY, ELEVENLABS_API_KEY, OPENAI_API_KEY, RUNWAY_API_KEY, HEYGEN_API_KEY, DID_API_KEY, SYNTHESIA_API_KEY

Project Management

Config
# Initialize project structure
qcut system project-init --directory ./my-project
# Preview file organization
qcut system project-organize --directory ./my-project --source ./incoming --dry-run
# Execute organization
qcut system project-organize --directory ./my-project --source ./incoming --recursive
# Check structure
qcut system project-info --directory ./my-project --json

Editor Commands

Control the running QCut editor via CLI. Requires QCut to be running. Most commands need --project-id.

Getting Started

Editor

Editor commands use the editor: prefix and communicate with QCut over HTTP (default: 127.0.0.1:8765).

# Check editor connectivity
qcut editor:health --json
# List projects and pick one
qcut editor:navigator:projects --json
# Use --project-id for most commands
qcut editor:timeline:info --project-id <id> --json
# Or use --resume for sticky sessions
qcut editor:timeline:info --resume my-session --json

editor:health

Check editor connectivity. Flags: --status-only, --deep

editor:console

List captured renderer console messages. Flags: --level, --since, --limit, --stream, --clear

editor:state:snapshot
