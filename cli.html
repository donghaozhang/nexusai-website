CLI v1.0

Pipeline CLI Reference

AI content generation, video analysis, transcription, YAML pipelines, and agentic video production — all from the command line.

$ bun run pipeline <command> [options]

Installation

The CLI is bundled with QCut. Three ways to run it:

# Dev (recommended)
bun run pipeline <command> [options]
# Direct source
bun run electron/native-pipeline/cli/cli.ts <command> [options]
# Production binary
qcut-pipeline <command> [options]

Quick Start

# Generate an image
bun run pipeline generate-image -t "A cinematic portrait at golden hour"
# Create a video
bun run pipeline create-video -m kling_2_6_pro -t "Ocean waves at sunset" -d 5s
# Transcribe audio
bun run pipeline transcribe -i audio.mp3 --srt
# Run a YAML pipeline
bun run pipeline run-pipeline -c pipeline.yaml -i "A sunset" --no-confirm
# Full video from an idea (ViMax)
bun run pipeline vimax:idea2video --idea "A detective in 1920s Paris" -d 120

Analysis Commands

Analyze, transcribe, query, and clip videos with AI.

Analysis

Analyze a video with AI vision.

FlagShortTypeDescription
--input-istringVideo file or URL required
--model-mstringVision model (default: gemini_qa)

transcribe

Analysis

Transcribe audio to text with optional SRT subtitle output.

bun run pipeline transcribe -i audio.mp3 --srt --language en
FlagShortTypeDescription
--input-istringAudio file or URL required
--model-mstringSTT model (default: scribe_v2)
--languagestringLanguage code (e.g. en, fr)
--srtbooleanGenerate .srt subtitle file
--srt-max-wordsintegerMax words per SRT block
--no-diarizebooleanDisable speaker diarization
--keytermsstring[]Domain keywords (repeatable)

query-video

Analysis

Query video segments for keep/cut analysis.

bun run pipeline query-video -i video.mp4 -t "Find all action scenes"
FlagShortTypeDescription
--input-istringVideo file or URL required
--promptstringQuery prompt
--text-tstringAlias for prompt
--model-mstringVision model (default: gemini_qa)

autoclip

Analysis

Extract highlight clips from a video using subtitle-based LLM analysis. 4-step pipeline: outline → timeline → scoring → ffmpeg cutting.

bun run pipeline autoclip -i video.mp4 -s subs.srt -o /tmp/clips --min-score 0.8
FlagShortTypeDescription
--input-istringInput video file required
--srt-file-sstringSRT/VTT subtitle file (auto-detects if omitted)
--output-ostringOutput directory for clips
--min-scorenumberMinimum score threshold 0-1 (default: 0.7)
--stepnumberRun only a specific step (1-4)
--dry-runbooleanRun analysis only, skip cutting

translate-video

Analysis

Translate a video's speech into another language using HeyGen Translate.

bun run pipeline translate-video -i video.mp4 -l Spanish
FlagShortTypeDescription
--input-istringVideo file or URL required
--language-lstringTarget language required
--output-ostringOutput directory
--audio-onlybooleanTranslate audio only (keep original video)
--speakersnumberNumber of speakers in the video

YAML Pipelines

Chain multiple AI steps into automated workflows.

run-pipeline

Pipeline

Run a multi-step YAML pipeline.

bun run pipeline run-pipeline -c pipeline.yaml -i "A sunset" --no-confirm
FlagShortTypeDescription
--config-cstringYAML pipeline file required
--input-istringInput text, file, or - for stdin
--save-intermediatesbooleanSave each step's output
--parallelbooleanParallel step execution
--max-workersintegerMax concurrent workers (default: 8)
--no-confirmbooleanSkip cost confirmation
--streambooleanJSONL progress on stderr

Pipeline YAML Schema

name: my-pipeline
steps:
  - type: text_to_image
    model: flux_dev
    params:
      image_size: landscape_16_9
    enabled: true
    retry_count: 0

  - type: parallel_group
    merge_strategy: COLLECT_ALL
    steps:
      - type: text_to_image
        model: flux_schnell

config:
  output_dir: ./output
  save_intermediates: true
  parallel: false
  max_workers: 8

Step types: text_to_image, image_to_image, text_to_video, image_to_video, video_to_video, avatar, motion_transfer, upscale, upscale_video, add_audio, text_to_speech, speech_to_text, image_understanding, prompt_generation

ViMax — Agentic Video Production

End-to-end AI video production: from idea to finished film.

vimax:idea2video

ViMax

Full pipeline: idea → screenplay → characters → portraits → storyboard → video.

bun run pipeline vimax:idea2video --idea "A detective in 1920s Paris" -d 120
FlagShortTypeDescription
--ideastringStory idea required
--duration-dstringTarget duration (seconds)
--no-portraitsbooleanSkip portrait generation
--no-referencesbooleanDisable character references
--config-cstringYAML config overrides
--llm-modelstringOverride LLM agent model
--image-modelstringOverride image model
--video-modelstringOverride video model

vimax:script2video

ViMax

Script → storyboard → video (from existing script.json).

bun run pipeline vimax:script2video --script script.json --portraits registry.json
FlagShortTypeDescription
--scriptstringScript JSON path required
--portraits-pstringPortrait registry JSON path
--no-referencesbooleanDisable character references

ViMax

--max-scenesintegerCap total scenes
--no-portraitsbooleanSkip portraits
--scripts-onlybooleanStop after scripts
--storyboard-onlybooleanStop after storyboard

estimate-cost

bun run pipeline estimate-cost -m veo3 -d 8s

API Key Management

Config

Keys stored in ~/.qcut/.env (mode 0600).

# Create .env template
bun run pipeline setup
# Set a key
bun run pipeline set-key --name FAL_KEY --value <key>
# Check all keys
bun run pipeline check-keys

Supported keys: FAL_KEY, GEMINI_API_KEY, OPENROUTER_API_KEY, ELEVENLABS_API_KEY, OPENAI_API_KEY, RUNWAY_API_KEY, HEYGEN_API_KEY, DID_API_KEY, SYNTHESIA_API_KEY

Project Management

Config
# Initialize project structure
bun run pipeline init-project --directory ./my-project
# Preview file organization
bun run pipeline organize-project --directory ./my-project --source ./incoming --dry-run
# Execute organization
bun run pipeline organize-project --directory ./my-project --source ./incoming --recursive
# Check structure
bun run pipeline structure-info --directory ./my-project --json

Progressive Help System

3-level structured help available with --help --json.

# Level 1: All commands
bun run pipeline --help --json
# Level 2: Command detail
bun run pipeline generate-image --help --json
# Level 3: Parameter detail
bun run pipeline generate-image --help model --json
