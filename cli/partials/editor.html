Editor Commands

Control the running QCut editor via CLI. Requires QCut to be running. Most commands need --project-id.

Getting Started

Editor

Editor commands use the editor: prefix and communicate with QCut over HTTP (default: 127.0.0.1:8765).

# Check editor connectivity
qcut editor:health --json
# List projects and pick one
qcut editor:navigator:projects --json
# Use --project-id for most commands
qcut editor:timeline:info --project-id <id> --json
# Or use --resume for sticky sessions
qcut editor:timeline:info --resume my-session --json

editor:health

Check editor connectivity. Flags: --status-only, --deep

editor:console

List captured renderer console messages. Flags: --level, --since, --limit, --stream, --clear

editor:errors

List captured renderer errors. Flags: --since, --limit, --stream, --clear

editor:undo / editor:redo

Undo or redo the last action.

editor:state:snapshot

Get editor state snapshot. Flags: --include (timeline, selection, playhead, media, editor, project)
