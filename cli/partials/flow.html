YAML Pipelines

Chain multiple AI steps into automated workflows.

flow run

Pipeline

Run a multi-step YAML pipeline.

qcut flow run -c pipeline.yaml -i "A sunset" --no-confirm
FlagShortTypeDescription
--config-cstringYAML pipeline file required
--input-istringInput text, file, or - for stdin
--save-intermediatesbooleanSave each step's output
--parallelbooleanParallel step execution
--max-workersintegerMax concurrent workers (default: 8)
--no-confirmbooleanSkip cost confirmation
--streambooleanJSONL progress on stderr

Pipeline YAML Schema

name: my-pipeline
steps:
  - type: text_to_image
    model: flux_dev
    params:
      image_size: landscape_16_9
    enabled: true
    retry_count: 0

  - type: parallel_group
    merge_strategy: COLLECT_ALL
    steps:
      - type: text_to_image
        model: flux_schnell

config:
  output_dir: ./output
  save_intermediates: true
  parallel: false
  max_workers: 8

Step types: text_to_image, image_to_image, text_to_video, image_to_video, video_to_video, avatar, motion_transfer, upscale, upscale_video, add_audio, text_to_speech, speech_to_text, image_understanding, prompt_generation

Flow — Agentic Video Production

End-to-end AI video production: from idea to finished film.

flow idea2video

Flow

Full pipeline: idea → screenplay → characters → portraits → storyboard → video.

qcut flow idea2video --idea "A detective in 1920s Paris" -d 120
FlagShortTypeDescription
--ideastringStory idea required
--duration-dstringTarget duration (seconds)
--no-portraitsbooleanSkip portrait generation
--no-referencesbooleanDisable character references
--config-cstringYAML config overrides
--llm-modelstringOverride LLM agent model
--image-modelstringOverride image model
--video-modelstringOverride video model

flow script2video

Flow

Script → storyboard → video (from existing script.json).

qcut flow script2video --script script.json --portraits registry.json
FlagShortTypeDescription
--scriptstringScript JSON path required
--portraits-pstringPortrait registry JSON path
--no-referencesbooleanDisable character references

flow novel2movie

Flow

Novel text file → chapter extraction → screenplay → video.

qcut flow novel2movie --novel book.txt --max-scenes 20
FlagShortTypeDescription
--novelstringNovel text file required
--titlestringOverride title
--max-scenesintegerCap total scenes
--no-portraitsbooleanSkip portraits
--scripts-onlybooleanStop after scripts
--storyboard-onlybooleanStop after storyboard

Flow — Portrait & Script Commands

Flow

flow script

Generate screenplay from an idea. Flags: --idea, --duration, --llm-model

flow characters

Extract character descriptions from text. Flags: --text, --llm-model

flow portraits

Generate character portraits. Flags: --text, --max-characters, --views, --image-model

flow storyboard

Generate storyboard images from script. Flags: --script, --portraits, --style

flow registry-create

Build portrait registry from directory. Flags: --input, --project-id

flow registry-show

Display registry contents. Flags: --input

system models

List ViMax-relevant models (image, video, image-to-video).
