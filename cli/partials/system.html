system login / logout

Auth

Sign in to route AI calls through QCut's license-server proxy and pay with credits. The returned JWT is stored as QCUT_AUTH_TOKEN in ~/.qcut/.env. Required for any qcut gen / qcut flow / qcut analyze command unless you've set a local provider key (e.g. GMI_API_KEY) for BYOK mode.

# Google OAuth (opens a browser window)
qcut system login
# Email + password (interactive)
qcut system login --email you@example.com
# Scripted / CI
qcut system login --email "$QCUT_EMAIL" --password "$QCUT_PASSWORD"
# Create a new account
qcut system signup --email you@example.com --password "$PW"
# Clear the stored token
qcut system logout

Credit balance: query the license server directly with your token —

curl -H "Authorization: Bearer $(qcut system get-key --name QCUT_AUTH_TOKEN --reveal)" \
    https://qcut-license-server.zdhpeter.workers.dev/api/credits/balance

Force proxy routing (ignore a local provider key for one call): env -u GMI_API_KEY qcut gen video -m gmi_veo31_lite_t2v -t "…"

system keys

Config

Keys stored in ~/.qcut/.env (mode 0600).

# Create .env template
qcut system setup
# Set a key
qcut system set-key --name FAL_KEY --value <key>
# Reveal a stored key (use --reveal for raw value)
qcut system get-key --name QCUT_AUTH_TOKEN --reveal
# Check all keys
qcut system check-keys

Supported keys: QCUT_AUTH_TOKEN, GMI_API_KEY, FAL_KEY, GEMINI_API_KEY, OPENROUTER_API_KEY, ANTHROPIC_API_KEY, ELEVENLABS_API_KEY, OPENAI_API_KEY, RUNWAY_API_KEY, HEYGEN_API_KEY, DID_API_KEY, SYNTHESIA_API_KEY

system project

Config
# Initialize project structure
qcut system project-init --directory ./my-project
# Preview file organization
qcut system project-organize --directory ./my-project --source ./incoming --dry-run
# Execute organization
qcut system project-organize --directory ./my-project --source ./incoming --recursive
# Check structure
qcut system project-info --directory ./my-project --json
